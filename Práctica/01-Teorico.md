Para la siguiente práctica puedes instalar pseint o utilizar una hoja y un papel, esto porque el punto primordial aquí es ir desarrollando la lógica de programación.

Para iniciar, vamos a realizar un "paso a paso" de una cosa que hagamos cotidianamente, como hornear un pastel, cambiar una llanta de un auto o similar. Esto se debe hacer de forma que quede bien claro qué debe hacer el usuario. A continuación te pondré un ejemplo de cómo se hace para que puedas guiarte: 

## Paso 1: Identificar el problema
Es muy importante que a la hora de programar, antes de comenzar a desarrollar el código, planteemos qué queremos que realice el programa y entender cómo resolverlo. En este caso nuestro "problema" es cocinar pancakes.

## Paso 2: Identificar las variables (o cosas a necesitar para la tarea)
Si bien aún no se ven qué son las variables, si sabemos que para cocinar pancakes necesitamos los siguientes elementos: 
- harina = 1 taza
- leche = 1 taza
- huevo = 1
- azucar = 2 cucharadas
- polvoHornear = 1 cucharadita
- aceite = 1 cucharada
- sartén
- cocina
- recipiente
- plato

## Paso 3: Secuencia de pasos
Ahora que ya tenemos nuestras variables o elementos a ocupar, vamos a realizar una secuencia de acciones que se deben seguir para poder realiza nuestros pancakes.
- Paso 1: Colocar la harina en un recipiente
- Paso 2: Agregar la leche
- Paso 3: Agregar el huevo
- Paso 4: Agregar el azúcar, polvo de hornear y aceite
- Paso 5: Mezclar todos los ingredientes
- Paso 6: Verificar si la mezcla es homogénea
- Paso 7: Prender la cocina
- Paso 8: Colocar el sartén en la cocina
- Paso 9: Cocinar la mezcla en el sartén
- Paso 10: Retirar el pancake del sartén y colocar en el plato
- Paso 11: Repetir pasos 9 y 10 hasta acabar mezcla

Y listo, con esto ya logramos desarrollar nuestro primer ejemplo de ejercicios para lograr desarrollar la lógica de programación con elementos que ya conocemos de nuestro día a día, sin la necesidad de aprender lenguajes de programación o similares (eso nos puede abrumar mucho al pricipio).
Otro punto importante a destacar es que no siempre se llegarán a las mismas soluciones que las otras personas, pero eso no quiere decir que una versión esté mal o sea mejor que la otra, muchas veces van a existir distintos caminos o formas de solucionar un mismo problema. Lo probable es que, tal vez, uno sea más óptimo que el otro, pero esto se soluciona según vayas mejorando tu lógica y conocimientos en programación, estructuras de datos y otros.
